## 协作流程

### 1. 开发流程
```
1. 玉桂狗编写代码 → 2. 本地 Xcode 运行测试 → 3. 截图发到飞书
   → 4. 昊昱确认 → 5. 提交到 GitHub → 6. 昊昱 Review → 7. 合并
```

### 2. 代码提交规范
- 每个功能开发完成后，在本地测试通过
- 截图展示功能效果（模拟器或真机）
- 昊昱确认后，提交到 GitHub
- 昊昱可以在 GitHub 上 Review 代码
- 确认无误后合并到 main 分支

### 3. 分支策略
- `main`: 稳定版本分支
- `develop`: 开发分支（可选）
- `feature/xxx`: 功能分支（大型功能）

### 4. 沟通方式
- 飞书：日常沟通、截图确认
- GitHub：代码 Review、Issue 跟踪
