# HSearch 项目协作流程

## 工作流程

```
提出修改 → 讨论设计方案 → 核对修改 → 拆解任务 → 运行修改 → 截图反馈 → 
双方确认 → 提交到 GitHub
```

## 各阶段职责

| 阶段 | 昊昱 | 玉桂狗 |
|------|------|--------|
| **提出修改** | ✅ 提出功能/bug 需求 | 倾听理解 |
| **讨论设计方案** | ✅ 确认方案方向 | ✅ 提供技术建议 |
| **核对修改** | ✅ 确认修改范围 | ✅ 明确具体改动点 |
| **拆解任务** | - | ✅ 拆分具体实施步骤 |
| **运行修改** | - | ✅ 编码 + 本地测试 |
| **截图反馈** | - | ✅ 模拟器截图发送 |
| **确认修改** | ✅ 看截图确认 | ✅ 自检通过 |
| **双方确认** | ✅ 最终确认 | ✅ 等待确认 |
| **提交 GitHub** | - | ✅ git commit + push |

## 截图确认标准

- ✅ **UI 改动**：截图展示前后对比
- ✅ **功能改动**：截图展示操作流程
- ✅ **Bug 修复**：截图证明问题已解决

## 提交 GitHub 前检查清单

- [ ] 代码运行正常
- [ ] 截图已发送并确认
- [ ] 双方确认无误
- [ ] commit message 清晰
